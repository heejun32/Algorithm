함수의 점근적 분석
    알고리즘의 효율성 분석
    이 강의에서는 일반적으로 1~2개의 변수만 있다고 가정함

    과정 1.
    알고리즘이 주어졌을 때, 무엇을 알고리즘의 성능을 테스트하는 지표로 삼을 것인가?

    과정 2.
    알고리즘과 연관된 데이터 구조를 고렿나다.

    과정 3. (매우 중요)
    알고리즘을 독립적인 환경에서 비교해야 한다.

    Leading Term: 차수가 높은 항이 가장 중요하다.
    그러나 계수(coefficients)역시 중요


    f(n) = a^kn^k ......
    g(n) = b^kn^k ......
    M = a^k/b^k + 1
    이때 n이 충분히 클 때 f(n) < Mg(n)이 된다.
    gn이 더 효율적인 알고리즘이라 하더라도 fn 을 구동시키는 컴퓨터가 m배 더 빠를대를 뜻함.
    lim (fn/gn) = c 면 이 둘은 비슷한 복잡도를 가진다고 생각된다. where 0 < C < 무한대

little O-Notation
    :타이트하게 알고리즘 복잡도 표시 (계층 살리기)
    for all n>=n0 and all constant C>0
    0<=fn<cgn
